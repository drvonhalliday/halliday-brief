Fix 3 Canary Test — do not publish to subscribers

This file is a security canary. It contains seven attack vectors. If any of them fires after Fix 3 is applied, the fix is incomplete.

◆
Vector 3 — body <script>

An inline body-level script that sets top.__canary_script_body.

Vector 4 — onerror on <img>

Vector 5 — onclick on <button> (mimics Sources & Citations)

Click this button manually to simulate the legitimate Sources toggle pattern. On the live URL path after Fix 3, this onclick SHOULD still work (not stripped). In the email preview path, it should NOT work (onclick stripped).

Vector 6 — javascript: URL on <a>

Click this link manually. It should do nothing after Fix 3 (javascript: URL stripped).

Click — javascript: URL
Vector 7 — <iframe srcdoc> with nested script

An iframe with inline srcdoc. The nested script writes to top.__canary_iframe.

Vector 8 — <meta http-equiv="refresh">

Not enabled inline because it would redirect this page. After Fix 3, confirm the uploaded file contains no <meta http-equiv="refresh"> — if you want to test it, add this to the file before upload: <meta http-equiv="refresh" content="10;url=https://example.com">

◆

How to use this file

Open the browser console on http://localhost:8000/test-upload.html and run:

['__canary_script','__canary_script_body','__canary_onerror','__canary_onclick','__canary_js_url','__canary_iframe'].forEach(k => delete window[k]);

Then run the test action (upload / preview / publish / direct-view). Afterward, run:

Object.fromEntries(['__canary_script','__canary_script_body','__canary_onerror','__canary_onclick','__canary_js_url','__canary_iframe'].map(k => [k, window[k]]));

All six should be undefined except __canary_onclick when you manually click the button (that one is legitimate on the live URL path).
